# PROMPT — Remove IsSimulated: true from Governance Handlers Confirmed Returning Real Data

## 1. Identification
- **Prompt ID:** P03.4
- **Phase:** 03-governance-finops-knowledge
- **Execution order:** 32
- **Type:** cleanup
- **Area:** Governance
- **Module:** governance
- **Bounded context:** governance-cleanup
- **Can run in parallel:** no (must run after P03.1, P03.2, P03.3 to confirm which handlers are real)
- **Risk level:** medium

## 2. Objective
Remove IsSimulated: true from all Governance handler responses that have been confirmed to return real data after P03.1 (Teams/Domains), P03.2 (remaining handlers), and P03.3 (FinOps). Update DemoBanner/SimulationIndicator frontend components to only display where data remains genuinely simulated. Produce a definitive summary documenting which handlers are now real, which remain simulated, and which are deferred.

## 3. Problem actual
The IsSimulated flag was intentionally introduced to honestly mark mock/fabricated data in Governance handler responses. After P03.1, P03.2, and P03.3 have replaced fabricated data with real cross-module queries, many handlers now set `IsSimulated = false` in their responses — but the property itself still exists on the response DTOs. Additionally, frontend DemoBanner components may still display simulation warnings even on pages where data is now real.

Current state (estimated post-P03.1/P03.2/P03.3):
- ~20-30 handlers now return real data (IsSimulated = false)
- ~20-30 handlers may still return simulated data (DEFERRED category from P03.2)
- Frontend DemoBanner/SimulationIndicator may not reflect the updated handler status

Evidence:
- `src/modules/governance/NexTraceOne.Governance.Application/Features/` — 58 feature directories with IsSimulated in response DTOs
- Frontend governance components in `src/frontend/` — DemoBanner, SimulationIndicator, or equivalent components
- P03.2 classification inventory — the definitive list of which handlers are CAN_REPLACE_NOW (now real), NEEDS_OWN_PERSISTENCE (now real), or DEFERRED (still simulated)

## 4. Resultado esperado
- All handlers confirmed returning real data no longer have `IsSimulated = true` in their response DTOs (property removed or always false).
- Frontend DemoBanner/SimulationIndicator components only display on governance pages where data remains genuinely simulated.
- Handlers still returning simulated data retain `IsSimulated = true` (do NOT prematurely remove).
- A definitive summary document lists: (a) handlers now real, (b) handlers still simulated with reason, (c) handlers deferred with blocking prompt reference.
- Grep count for `IsSimulated = true` in governance handlers decreases by ≥20 compared to pre-P03.1 baseline.

## 8. Passos obrigatórios
1. **Establish baseline** — Run `grep -rc "IsSimulated = true" src/modules/governance/NexTraceOne.Governance.Application/Features/ | awk -F: '{s+=$2} END {print s}'` to count current IsSimulated = true occurrences. Record this number.
2. **Compare P03.2 handler inventory** — Identify all handlers where P03.1/P03.2/P03.3 set `IsSimulated = false` or where real data now flows. These are candidates for IsSimulated removal.
3. **Remove IsSimulated = true from confirmed-real handlers** — For each handler confirmed returning real data:
   - Remove `IsSimulated = true` from the response object construction
   - If the response DTO has `bool IsSimulated` property, either: (a) remove the property entirely if no longer needed, or (b) set it to `false` permanently
   - Prefer option (a) removal if no frontend code depends on it for that specific handler
4. **Update frontend DemoBanner/SimulationIndicator** — Search frontend source for components that check IsSimulated:
   - Identify which governance pages show simulation banners
   - Update logic to only show simulation banner when the specific page's API response still contains `IsSimulated = true`
   - Remove simulation banner from pages backed by now-real handlers
5. **Leave IsSimulated intact for still-simulated handlers** — For handlers classified as DEFERRED in P03.2 inventory, do NOT remove IsSimulated. They still return simulated data.
6. **Create definitive summary** — Produce a summary listing:
   - **Handlers now real** (IsSimulated removed): handler name, data source
   - **Handlers still simulated** (IsSimulated retained): handler name, reason, blocking prompt reference
   - **Handlers deferred** (IsSimulated retained with TODO): handler name, specific dependency
7. **Run grep count verification** — Run same grep as step 1. Count must decrease by ≥20.
8. **Update tests** — Modify unit tests that assert `IsSimulated = true` on now-real handlers to assert property removed or `IsSimulated = false`.

## 13. Riscos e cuidados
- **Premature removal:** The biggest risk is removing IsSimulated from a handler that still returns fabricated data. Always cross-reference with P03.2 classification inventory before removing.
- **Frontend regression:** Removing IsSimulated from response DTOs may break frontend components that conditionally render based on this property. Verify frontend usage before removing the DTO property itself.
- **Incomplete P03.2 inventory:** If P03.2 didn't produce a clear inventory, this prompt needs to independently verify each handler's data source before proceeding.
- **Test fragility:** Tests that assert specific IsSimulated values will break when the property is removed. Update tests in the same commit.
- **DemoBanner logic complexity:** If DemoBanner uses a global simulated flag rather than per-page checking, the update may be more complex than expected.
