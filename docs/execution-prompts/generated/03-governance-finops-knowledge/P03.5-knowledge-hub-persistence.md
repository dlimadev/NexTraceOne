## 13. Riscos e cuidados
- **Migration not applied:** If P02.8f migration hasn't been applied to the development database, integration tests will fail. Ensure migration is applied before running integration tests.
- **FTS configuration complexity:** PostgreSQL FTS with EF Core requires specific Npgsql provider methods. If the FTS implementation uses raw SQL instead of EF methods, ensure SQL injection prevention.
- **GIN index performance:** Without a GIN index on the ts_vector column, FTS queries will do sequential scans. Verify index exists in migration.
- **Integration test database dependency:** Integration tests need a running PostgreSQL instance. Use TestContainers or an in-memory test database if available in the test infrastructure.
- **Repository SaveChangesAsync:** If repositories use a unit-of-work pattern, verify that SaveChangesAsync is called after AddAsync. Missing SaveChangesAsync is a common persistence bug.
