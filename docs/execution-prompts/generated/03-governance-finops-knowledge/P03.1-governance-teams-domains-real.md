# PROMPT — Replace IsSimulated Teams/Domains Governance Handlers with Real Cross-Module Data

## 1. Identification
- **Prompt ID:** P03.1
- **Phase:** 03-governance-finops-knowledge
- **Execution order:** 27
- **Type:** feature-implement
- **Area:** Governance
- **Module:** governance
- **Bounded context:** teams-domains
- **Can run in parallel:** no (modifies governance handlers shared with P03.2 flow)
- **Risk level:** medium

## 2. Objective
Replace IsSimulated Teams/Domains handlers in the Governance module with real cross-module data sourced from IdentityAccess (teams, users, roles) and Catalog (services, domains, dependencies). After this prompt, all Teams and Domains governance handlers must return genuine data retrieved via established module interfaces and read-only cross-module queries instead of hardcoded fabricated responses.

## 3. Problem actual
Per forensic audit: "Governance module: 100% MOCK by design — 74 handlers return IsSimulated: true. GovernanceDbContext has 8 entities but no handlers read from other modules. Teams exist in IdentityDbContext, services/domains exist in CatalogGraphDbContext."

Specifically for Teams/Domains:
- `ListTeams`, `GetTeamDetail`, `GetTeamGovernanceSummary`, `GetCrossTeamDependencies` handlers fabricate team data instead of querying IdentityDbContext where real teams reside.
- `ListDomains`, `GetDomainDetail`, `GetDomainGovernanceSummary`, `GetCrossDomainDependencies` handlers fabricate domain data instead of querying CatalogGraphDbContext where service assets grouped by domain reside.
- `CreateTeam`, `UpdateTeam`, `CreateDomain`, `UpdateDomain` may persist to GovernanceDbContext but should coordinate with IdentityAccess and Catalog respectively.

Evidence paths:
- `src/modules/governance/NexTraceOne.Governance.Application/Features/GetTeamDetail/GetTeamDetail.cs` — uses ITeamRepository and ICatalogGraphModule but may still have fabricated fallbacks
- `src/modules/governance/NexTraceOne.Governance.Application/Features/ListTeams/ListTeams.cs` — lists teams with potentially simulated data
- `src/modules/identityaccess/NexTraceOne.IdentityAccess.Infrastructure/Persistence/IdentityDbContext.cs` — real team/user data source
- `src/modules/catalog/NexTraceOne.Catalog.Infrastructure/Graph/Persistence/CatalogGraphDbContext.cs` — real service/domain data source

## 4. Resultado esperado
- All Teams-related governance handlers (`ListTeams`, `GetTeamDetail`, `GetTeamGovernanceSummary`, `GetCrossTeamDependencies`, `CreateTeam`, `UpdateTeam`) retrieve real data from IdentityAccess module interfaces or IdentityDbContext read queries.
- All Domains-related governance handlers (`ListDomains`, `GetDomainDetail`, `GetDomainGovernanceSummary`, `GetCrossDomainDependencies`, `CreateDomain`, `UpdateDomain`) retrieve real data from Catalog module interfaces (IContractsModule / ICatalogGraphModule) or CatalogGraphDbContext read queries.
- Affected handlers return `IsSimulated = false` in their response DTOs.
- No hardcoded/fabricated team names, user counts, domain names, or service counts remain in these handlers.
- Unit tests confirm real data retrieval paths are exercised.
- Cross-module boundaries respected: no direct writes to IdentityAccess or Catalog domain layers.
