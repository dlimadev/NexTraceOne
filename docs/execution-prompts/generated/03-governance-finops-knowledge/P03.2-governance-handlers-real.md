## 13. Riscos e cuidados
- **High complexity:** Modifying 40+ handlers in a single prompt is ambitious. Prioritize CAN_REPLACE_NOW handlers first, then NEEDS_OWN_PERSISTENCE.
- **Cross-module interface completeness:** Some interfaces from P02.2/P02.3 may not expose all needed methods. Document gaps rather than fabricating workarounds.
- **GovernanceDbContext entity additions:** Adding new entities without a migration will cause runtime errors. Ensure migration is planned (or confirm it already exists).
- **Response DTO stability:** Changing from fabricated to real data may change response shapes. Keep DTO contracts stable even if underlying data source changes.
- **Performance impact:** Multiple cross-module queries per handler may degrade performance. Use async/await, CancellationToken, and consider caching patterns for hot paths.
- **Test coverage:** With 40+ handlers, comprehensive unit testing is essential but time-consuming. Focus on handlers most critical to the governance product pillar.
