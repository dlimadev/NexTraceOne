## 2. Objective

## 13. Riscos e cuidados
- **ICostIntelligenceModule method availability:** P02.4 must have implemented sufficient methods on ICostIntelligenceModule. If some methods are missing (e.g., GetCostByDomainAsync), document the gap and use available methods with manual grouping.
- **Empty CostIntelligenceDbContext:** If no cost data has been ingested, all FinOps handlers will return zero/empty results. This is correct behavior — do NOT fabricate fallback data for production handlers. Seed data is only for development.
- **Performance of cost aggregations:** Aggregating cost records across services/teams/domains may be slow with large datasets. Use efficient queries, consider pagination, pass CancellationToken.
- **FinOps contextual requirement:** NexTraceOne FinOps is explicitly NOT a generic cost dashboard. Every cost figure must be traceable to a service, team, domain, or environment. Do not show orphan cost data.
- **Mapping complexity:** Governance FinOps DTOs may have different shapes than CostIntelligence module responses. Mapping must preserve all semantic meaning.
