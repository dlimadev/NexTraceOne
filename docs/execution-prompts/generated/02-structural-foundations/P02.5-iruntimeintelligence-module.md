## 2. Objective
Implement the `IRuntimeIntelligenceModule` cross-module interface defined in `NexTraceOne.OperationalIntelligence.Contracts/Runtime/ServiceInterfaces/IRuntimeIntelligenceModule.cs`. The implementation must read real data from `RuntimeIntelligenceDbContext` (which has an InitialCreate migration at `20260325210556`). This interface enables other bounded contexts вЂ” particularly Catalog (service health display), ChangeGovernance (pre-change health baseline), and AIKnowledge (health-aware AI analysis) вЂ” to query runtime health and observability data for services.

## 3. Problem actual
The `IRuntimeIntelligenceModule` interface is defined with two methods but has zero implementation:

```csharp
public interface IRuntimeIntelligenceModule
{
    Task<string?> GetCurrentHealthStatusAsync(string serviceName, string environment, CancellationToken ct = default);
    Task<decimal?> GetObservabilityScoreAsync(string serviceName, string environment, CancellationToken ct = default);
}
```

`RuntimeIntelligenceDbContext` has entities: `RuntimeSnapshot` (health/performance snapshots), `RuntimeBaseline` (baseline metrics for drift comparison), `DriftFinding` (detected drift between baselines and current state). The `DriftDetectionJob` in BackgroundWorkers already runs to populate drift findings.

Without this implementation:
- Service detail pages cannot show real health status
- Change confidence scoring cannot factor in pre-change health baseline
- AI grounding cannot include runtime context
- Service catalog reliability views lack real data
