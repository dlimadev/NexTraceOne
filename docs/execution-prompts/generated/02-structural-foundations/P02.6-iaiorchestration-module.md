## 2. Objective
Implement the `IAiOrchestrationModule` cross-module interface defined in `NexTraceOne.AIKnowledge.Contracts/Orchestration/ServiceInterfaces/IAiOrchestrationModule.cs`. Currently the interface is empty (no methods defined). This prompt requires first defining the interface methods based on the `AiOrchestrationDbContext` entities and cross-module consumption needs, then implementing them. The interface enables other modules — particularly Governance (AI usage reports), OperationalIntelligence (AI-assisted investigation context), and Audit (AI interaction audit trail) — to query AI conversation and orchestration data without referencing the AIKnowledge DbContext directly.

## 3. Problem actual
The `IAiOrchestrationModule` interface exists but is empty — no methods are defined:

```csharp
public interface IAiOrchestrationModule
{
    // Empty — planned but not designed
}
```

`AiOrchestrationDbContext` has entities: `AiContext` (aggregated analysis contexts), `AiConversation` (multi-turn conversations), `GeneratedTestArtifact` (AI-generated test artifacts), and `KnowledgeSuggestion` (AI-suggested knowledge entries). After P01.7 (conversation persistence), conversations will have real persisted data. Other modules need to query this data for:
- Executive AI usage reports
- AI interaction audit trails
- Agent execution result summaries
- Conversation context for service-specific AI analysis

## 4. Resultado esperado

- `AiOrchestrationModule` class implementing all methods, reading from `AiOrchestrationDbContext`
