# PROMPT — Implement IChangeIntelligenceModule cross-module interface with real data from ChangeIntelligenceDbContext

## 1. Identification
- **Prompt ID:** P02.3
- **Phase:** 02-structural-foundations
- **Execution order:** 22
- **Type:** feature-implement
- **Area:** backend
- **Module:** changegovernance
- **Bounded context:** change-interface
- **Can run in parallel:** yes (with P02.2 — group-D)
- **Risk level:** medium

## 2. Objective
Implement the `IChangeIntelligenceModule` cross-module interface defined in `NexTraceOne.ChangeGovernance.Contracts/ChangeIntelligence/ServiceInterfaces/IChangeIntelligenceModule.cs`. The implementation must read real data from `ChangeIntelligenceDbContext` (which has working migrations and domain entities) and be registered in DI. This interface enables other bounded contexts — particularly OperationalIntelligence (incident↔change correlation), AIKnowledge (AI grounding with change context), and Governance (risk assessment) — to query change/release data without violating bounded context boundaries.

## 3. Problem actual
The `IChangeIntelligenceModule` interface is defined with three methods and two DTOs but has zero implementation:

```csharp
public interface IChangeIntelligenceModule
{
    Task<ReleaseDto?> GetReleaseAsync(Guid releaseId, CancellationToken cancellationToken);
    Task<decimal?> GetChangeScoreAsync(Guid releaseId, CancellationToken cancellationToken);
    Task<BlastRadiusDto?> GetBlastRadiusAsync(Guid releaseId, CancellationToken cancellationToken);
}
```

DTOs `ReleaseDto` and `BlastRadiusDto` are defined alongside the interface in the Contracts project. `ChangeIntelligenceDbContext` exists at `src/modules/changegovernance/NexTraceOne.ChangeGovernance.Infrastructure/ChangeIntelligence/Persistence/ChangeIntelligenceDbContext.cs` with InitialCreate migration.

Without this implementation:
- P01.1 (incident↔change correlation) cannot query change data across modules cleanly
- P03.2 (change governance workflows) cannot proceed
- AI grounding for change intelligence context is blocked
- Blast radius analysis cannot be queried from other modules

## 4. Resultado esperado
- A `ChangeIntelligenceModule` class implementing `IChangeIntelligenceModule` in the ChangeGovernance Infrastructure project
- `GetReleaseAsync` reads a release entity from ChangeIntelligenceDbContext and maps to `ReleaseDto`
- `GetChangeScoreAsync` returns the calculated change confidence score for a release
- `GetBlastRadiusAsync` returns blast radius data (affected services, risk level, scope) for a release
- All queries are read-only with `AsNoTracking()`
- Registration in DI via the ChangeIntelligence module's DependencyInjection
- Verification that both ApiHost and BackgroundWorkers call the module registration
