## 2. Objective
Generate the EF Core `InitialCreate` migration for `ProductAnalyticsDbContext`. This DbContext has **no existing migrations** — the `Migrations/` directory is empty. The migration must be created from scratch, respecting the `pan_` table prefix convention per `docs/architecture/database-table-prefixes.md`. The ProductAnalytics module stores analytics events extracted from the former GovernanceDbContext in P2.3.

## 3. Problem actual
`ProductAnalyticsDbContext` exists at `src/modules/productanalytics/NexTraceOne.ProductAnalytics.Infrastructure/Persistence/ProductAnalyticsDbContext.cs` with one entity:
- `AnalyticsEvent` — product analytics events (extracted from GovernanceDbContext in P2.3)

The `Migrations/` directory at `src/modules/productanalytics/NexTraceOne.ProductAnalytics.Infrastructure/Persistence/Migrations/` is empty. The DbContext already has:
- `ConfigurationsAssembly` override pointing to `typeof(ProductAnalyticsDbContext).Assembly`
- `OutboxTableName` override set to `"pan_outbox_messages"`

Entity type configurations should use the `pan_` prefix for table names.

Additionally, the ProductAnalytics module infrastructure is NOT registered in `BackgroundWorkers/Program.cs`, meaning outbox processing cannot start until P02.1 adds it.

## 4. Resultado esperado
- InitialCreate migration generated for ProductAnalyticsDbContext

- Outbox table named `pan_outbox_messages` with `idempotency_key` unique index
- Migration applies cleanly to a fresh database
- Model snapshot generated and consistent
