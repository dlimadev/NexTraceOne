# PROMPT — Add code coverage tracking and enforcement thresholds

## 1. Identification
- **Prompt ID:** P05.2
- **Phase:** 05-quality-hardening
- **Execution order:** 40
- **Type:** pipeline-config
- **Area:** ci-cd
- **Module:** platform
- **Bounded context:** coverage
- **Can run in parallel:** yes (group-E)
- **Risk level:** low

## 2. Objective
Add code coverage collection and threshold enforcement to the CI pipeline for both backend (.NET) and frontend (Vitest) test suites. Establish baseline coverage thresholds that can be gradually increased.

## 3. Problem actual
Per forensic audit (`tests-quality-pipelines-report.md`): "Quality gates missing: coverage tracking." The project has 1,447 backend tests and 264 frontend tests, but coverage percentage is unknown and not enforced. There is no mechanism to detect coverage regression.

## 4. Resultado esperado
- Backend test run produces Cobertura XML coverage report
- Frontend test run produces lcov coverage report
- CI pipeline enforces minimum thresholds: backend ≥ 40%, frontend ≥ 30% (starting thresholds)
- Coverage reports uploaded as CI artifacts
- Clear failure message when coverage drops below threshold

## 5. Escopo permitido
- **Directories:** `.github/workflows/`, test project directories
- **Files:** `ci.yml`, test `.csproj` files (coverlet package), `vitest.config.ts`
- **Artifact types:** Workflow YAML, project files, config files

## 6. Escopo proibido
- Do NOT modify test code or test assertions
- Do NOT modify application source code
- Do NOT modify other workflows (e2e, staging, production)
- Do NOT add external coverage services (Codecov, Coveralls) — keep artifacts local
- Do NOT lower existing test count

## 7. Ficheiros principais candidatos a alteração
1. `.github/workflows/ci.yml` — add coverage collection and threshold check steps
2. `tests/modules/*/NexTraceOne.*.Tests.csproj` — add `coverlet.collector` package reference if missing
3. `src/frontend/vitest.config.ts` — add coverage configuration
4. `src/frontend/package.json` — add `@vitest/coverage-v8` if not present
5. Documentation noting coverage thresholds and how to increase them

## 8. Passos obrigatórios
1. **Backend coverage setup:**
   - Verify `coverlet.collector` NuGet package exists in test projects; add to those missing it
   - In ci.yml backend test step: add `--collect:"XPlat Code Coverage" --results-directory ./coverage`
   - Add step to merge coverage reports: `dotnet tool install -g dotnet-reportgenerator-globaltool && reportgenerator -reports:./coverage/**/coverage.cobertura.xml -targetdir:./coverage-report -reporttypes:Cobertura`
   - Add step to check threshold: parse Cobertura XML for line coverage percentage, fail if < 40%
2. **Frontend coverage setup:**
   - In `vitest.config.ts`: add `coverage: { provider: 'v8', reporter: ['lcov', 'text-summary'], thresholds: { lines: 30 } }`
   - In ci.yml frontend test step: add `--coverage` flag
3. **Upload artifacts:**
   - Add `actions/upload-artifact` step for coverage reports
4. **Threshold enforcement:**
   - Backend: use shell script to parse Cobertura XML line rate attribute
   - Frontend: Vitest `thresholds` will fail automatically if below threshold
5. Document thresholds and how to update them

## 9. Responsabilidades permitidas

## 10. Responsabilidades proibidas
