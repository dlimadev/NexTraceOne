# PROMPT — Make E2E Playwright tests a mandatory gate on PR merges

## 1. Identification
- **Prompt ID:** P05.1
- **Phase:** 05-quality-hardening
- **Execution order:** 39
- **Type:** pipeline-config
- **Area:** ci-cd
- **Module:** platform
- **Bounded context:** e2e-gate
- **Can run in parallel:** no
- **Risk level:** medium

## 2. Objective
Configure the CI/CD pipeline so that Playwright E2E tests run on every pull request and block merge on failure. Currently E2E tests run nightly only and do not gate PR merges.

## 3. Problem actual
Per forensic audit (`tests-quality-pipelines-report.md`): "e2e.yml: PARTIAL — nightly only, doesn't block PRs. Quality gates missing: E2E gate on PRs." This means broken core flows (Incidents, AI Assistant, Catalog, Changes) can merge into main without E2E validation. The 8 Playwright specs and 5 real-env tests exist but only run on schedule.

## 4. Resultado esperado
- E2E test job runs on every pull_request event targeting main
- PR merge is blocked if any E2E test fails
- E2E job builds backend, starts API server, builds frontend, runs Playwright
- Timeout configured to prevent hanging (15 minutes max)
- E2E results visible in PR checks

## 5. Escopo permitido
- **Directories:** `.github/workflows/`
- **Files:** `ci.yml`, `e2e.yml`, possibly `playwright.config.ts` (timeout only)
- **Artifact types:** GitHub Actions workflow YAML, documentation

## 6. Escopo proibido
- Do NOT modify existing Playwright test specs
- Do NOT modify application source code
- Do NOT modify other workflows (staging.yml, production.yml, security.yml)
- Do NOT modify frontend or backend code
- Do NOT add new test frameworks

## 7. Ficheiros principais candidatos a alteração
1. `.github/workflows/ci.yml` — add E2E job or reference
2. `.github/workflows/e2e.yml` — add `pull_request` trigger alongside schedule
3. `playwright.config.ts` — adjust timeout if needed for CI environment
4. `docs/architecture/` — document the new E2E gate

## 8. Passos obrigatórios
1. Open `.github/workflows/e2e.yml` and examine current trigger (should be `schedule` only)
2. Add `pull_request: branches: [main]` trigger alongside existing schedule trigger
3. Ensure the E2E job includes: checkout, .NET setup, Node.js setup, backend build + start, frontend build, Playwright install, Playwright run
4. Add `timeout-minutes: 15` to the E2E job
5. Configure the job to use a test database (verify existing DB configuration in e2e workflow)
6. Ensure the job name matches what branch protection rules expect (e.g., "e2e-tests")
7. In `.github/workflows/ci.yml`, add `needs: [e2e]` if E2E is a separate job, OR add a `required` status check note
8. Document: "To enforce as required check, repository admin must add 'e2e-tests' to branch protection required status checks"
9. Test locally: verify the workflow YAML is valid with `act` or manual inspection

## 9. Responsabilidades permitidas
- Modify workflow trigger configuration
- Add E2E job to CI pipeline
- Adjust timeouts
- Add documentation

## 10. Responsabilidades proibidas
- Do NOT modify test spec content
- Do NOT modify application code
- Do NOT change existing test assertions
- Do NOT add new dependencies to the project

## 11. Critérios de aceite
- [ ] `.github/workflows/e2e.yml` triggers on `pull_request` targeting main
- [ ] E2E job includes backend build + API server start + frontend build + Playwright run
- [ ] `timeout-minutes: 15` configured
- [ ] Job uses test database (not production)
- [ ] Workflow YAML is syntactically valid
- [ ] Documentation explains how to enable as required check in branch protection

## 12. Validações obrigatórias
- Workflow YAML passes yamllint or similar validation
- No changes to application source code
- No changes to test spec files
- Existing CI workflow still functions correctly

## 13. Riscos e cuidados
- MEDIUM RISK: E2E tests may be flaky — consider adding `--retries 1` to Playwright command
- CAUTION: Backend server startup time — ensure health check wait before running tests
- CAUTION: Test database must be isolated — verify no shared state with other CI jobs

## 14. Dependências
- **Depends on:** P01.3 (incidents frontend real), P01.8 (AI frontend real) — E2E needs real pages, not mock
- **Unblocks:** Gate 4 closure (quality hardened)
- **Blocked until done:** Gate 4 cannot close

## 15. Entregáveis esperados

- Documentation on enabling required status check
- Verified workflow YAML validity
