# PROMPT — Validate all EF Core migrations apply cleanly in CI

## 1. Identification
- **Prompt ID:** P05.5
- **Phase:** 05-quality-hardening
- **Execution order:** 43
- **Type:** pipeline-config
- **Area:** ci-cd
- **Module:** platform
- **Bounded context:** migration-gate
- **Can run in parallel:** no
- **Risk level:** medium

## 2. Objective
Add a CI job that starts a fresh PostgreSQL database and runs all EF Core migrations for every DbContext, verifying they apply without errors. This prevents broken migrations from reaching staging/production.

## 3. Problem actual
After P02.8a-g adds 7 new migrations (RuntimeIntelligence, CostIntelligence, AiOrchestration, ExternalAI, Integrations, Knowledge, ProductAnalytics), the total reaches ~22+ DbContexts with migrations. There is no automated CI step to verify all migrations apply cleanly. A broken migration discovered only at deployment time could cause significant downtime.

## 4. Resultado esperado
- CI job starts PostgreSQL container
- Runs `dotnet ef database update` for all DbContexts
- Verifies zero error exit code
- Verifies tables were created (basic sanity check)
- Fails the CI pipeline if any migration fails

## 5. Escopo permitido
- **Directories:** `.github/workflows/`, scripts/
- **Files:** `ci.yml`, migration validation script
- **Artifact types:** Workflow YAML, shell script

## 6. Escopo proibido
- Do NOT modify migration files
- Do NOT modify DbContext code
- Do NOT modify application source code
- Do NOT modify existing CI jobs
- Do NOT modify staging/production workflows

## 7. Ficheiros principais candidatos a alteração
1. `.github/workflows/ci.yml` — add migration validation job
2. `scripts/validate-migrations.sh` — script to run all migrations
3. `docs/architecture/` — document migration validation process
4. Possibly a `docker-compose.ci.yml` for the test database

## 8. Passos obrigatórios
1. Create `scripts/validate-migrations.sh` that:
   - Sets `DATABASE_URL` to the test PostgreSQL instance
   - Lists all DbContext names and their Infrastructure project paths:
     ```
     IdentityDbContext → src/modules/identityaccess/NexTraceOne.IdentityAccess.Infrastructure/
     AuditDbContext → src/modules/auditcompliance/NexTraceOne.AuditCompliance.Infrastructure/
     CatalogGraphDbContext → src/modules/catalog/NexTraceOne.Catalog.Infrastructure/Graph/
     ContractsDbContext → src/modules/catalog/NexTraceOne.Catalog.Infrastructure/Contracts/
     DeveloperPortalDbContext → src/modules/catalog/NexTraceOne.Catalog.Infrastructure/DeveloperPortal/
     ChangeIntelligenceDbContext → src/modules/changegovernance/NexTraceOne.ChangeGovernance.Infrastructure/
     IncidentDbContext → src/modules/operationalintelligence/NexTraceOne.OperationalIntelligence.Infrastructure/
     GovernanceDbContext → src/modules/governance/NexTraceOne.Governance.Infrastructure/
     AiGovernanceDbContext → src/modules/aiknowledge/NexTraceOne.AIKnowledge.Infrastructure/Governance/
     NotificationDbContext → src/modules/notifications/NexTraceOne.Notifications.Infrastructure/
     ConfigurationDbContext → src/modules/configuration/NexTraceOne.Configuration.Infrastructure/
     + 7 new contexts from P02.8a-g
     ```
   - Runs `dotnet ef database update --context <ContextName> --project <InfraProject> --startup-project src/platform/NexTraceOne.ApiHost/` for each
   - Verifies exit code 0 for each
   - After all migrations: runs `psql -c "SELECT count(*) FROM information_schema.tables WHERE table_schema = 'public'"` and verifies count > 0
2. In ci.yml: add a job `validate-migrations` that:
   - Uses `services: postgres: image: postgres:16` service container
   - Runs .NET restore and build
   - Runs `scripts/validate-migrations.sh`
3. Configure the job to run after unit tests pass (to not waste CI time on migration checks if tests fail)
4. Add `timeout-minutes: 10` to prevent hanging
5. Make the job non-blocking initially (`continue-on-error: true`) with a comment to make it blocking after P02.8a-g completes

## 9. Responsabilidades permitidas
- Create migration validation script

## 10. Responsabilidades proibidas

- [ ] Documentation lists all validated DbContexts

## 13. Riscos e cuidados

- CAUTION: Some DbContexts share the same database but different schemas — ensure no conflicts
- CAUTION: EF tools must be installed (`dotnet tool install --global dotnet-ef`)
- CAUTION: Start as `continue-on-error: true` until all 7 new migrations from P02.8a-g exist

## 14. Dependências

- **Unblocks:** Database deployment confidence
- **Blocked until done:** Nothing (starts non-blocking)
