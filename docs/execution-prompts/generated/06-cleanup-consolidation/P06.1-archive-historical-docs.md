# PROMPT — Archive historical execution reports and legacy documentation to docs/archive/

## 1. Identification
- **Prompt ID:** P06.1
- **Phase:** 06-cleanup-consolidation
- **Execution order:** 45
- **Type:** cleanup
- **Area:** documentation
- **Module:** cross-cutting

## 2. Objective
Move historical execution reports (e14–e18 series), p0/p1 security hardening reports, 11-review-modular suite, and earlier audit/trail documents into `docs/archive/` with proper index files. This reduces documentation clutter in the active workspace while preserving every document for git-history reference and future audits.

## 3. Problem actual
Per the audit `remove-archive-consolidate-report.md`: "ARCHIVE_CANDIDATE: ~100+ historical documents (e14-e18 execution reports, p0-p1 security hardening reports, 11-review-modular suite, earlier audit trails)." These documents have genuine historical value — they record decisions, findings, and rationale from past phases — but they clutter the active documentation directories and confuse contributors about which docs represent current system state. Without archival, new team members cannot distinguish current guidance from superseded reports.

Evidence paths:
- `docs/archive/old-phase-reports/` — some archival already started but is incomplete
- `docs/archive/old-execution-plans/` — partial archival exists
- `docs/archive/old-roadmaps/` — partial archival exists
- `docs/audit-forensic-2026-03/` — forensic audit (must NOT be moved; it is active reference)

## 4. Resultado esperado
- A well-organized `docs/archive/` directory tree with clear subdirectories per category: `execution-reports/`, `security-hardening/`, `review-modular/`, `trails/`
- A `docs/archive/README.md` that explains the purpose, structure, and policy of the archive
- All identified historical documents relocated — zero deletions
- Any in-repo markdown cross-references that pointed at moved files are updated to their new locations
- Active documentation directories contain only current, authoritative documents
- No build or test references broken by the move

## 5. Escopo permitido
- **Directories:** `docs/` (all subdirectories except `docs/audit-forensic-2026-03/`)
- **Operations:** file moves (git mv), new directory creation, new README/index files, reference link updates in existing markdown files
- **Artifact types:** Markdown documentation files, directory structure

## 6. Escopo proibido
- Do NOT delete any files — this is a move-only operation
- Do NOT modify `docs/audit-forensic-2026-03/` in any way — forensic audit is active reference
- Do NOT modify any source code files under `src/`
- Do NOT modify test files under `tests/`
- Do NOT modify configuration or infrastructure files
- Do NOT modify `docs/PRODUCT-VISION.md`, `docs/ARCHITECTURE-OVERVIEW.md`, or any current strategic docs
