## 2. Objective
Replace the hardcoded `mockConversations` in `AiAssistantPage.tsx` with real API calls using TanStack Query. Implement SSE (Server-Sent Events) consumption for streaming AI responses via `POST /api/v1/ai/chat/stream`. Connect to conversation list and detail endpoints. Add proper loading, error, and empty states with i18n.

## 3. Problem actual
Per forensic audit: "AiAssistantPage.tsx uses mockConversations hardcoded with comment: 'AssistantPanel has mock fallback when backend fails'." The AI assistant page is one of the critical broken flows. It shows fabricated conversation data that cannot reflect real AI interactions. The `AssistantPanel.tsx` component also contains mock fallback logic. After P01.6 (real LLM) and P01.7 (conversation persistence), the backend will serve real data — the frontend must consume it.

- SSE streaming implemented: assistant response streams token-by-token into the chat UI
- TanStack Query manages server state with proper caching and invalidation
- Loading, error, and empty states rendered with i18n keys
- Error states cover: LLM unavailable, rate limited, no budget remaining, network error
- Zero references to mockConversations in the codebase

## 8. Passos obrigatórios
1. Create or update `src/frontend/src/features/ai-hub/api/aiChat.ts`:
   - `fetchConversations()`: GET `/api/v1/ai/conversations` → returns list of conversations with metadata
   - `fetchConversationById(id: string)`: GET `/api/v1/ai/conversations/{id}` → returns conversation with all turns
   - `sendMessage(payload: SendMessagePayload)`: POST `/api/v1/ai/chat` → sends message, returns complete response
   - `streamMessage(payload: SendMessagePayload, onChunk: (chunk: string) => void)`: POST `/api/v1/ai/chat/stream` → SSE streaming, calls onChunk per token
   - Use axios instance with auth headers (follow existing patterns in `ai-hub/api/aiGovernance.ts`)
2. Implement SSE consumption for streaming:
   - Use `EventSource` or `fetch` with `ReadableStream` for SSE
   - Handle stream events: `message` (token chunk), `done` (stream complete), `error` (stream error)
   - Accumulate streamed tokens into the assistant message in real-time
   - Handle stream interruption gracefully
3. In `AiAssistantPage.tsx`:
   - Remove the `mockConversations` array entirely
   - Add `useQuery({ queryKey: ['ai-conversations'], queryFn: fetchConversations })` for conversation list
   - Add `useQuery({ queryKey: ['ai-conversation', id], queryFn: () => fetchConversationById(id) })` for detail
   - Add `useMutation` for sending messages (with optimistic update for user message)
   - Add proper loading state: `if (isLoading) return <LoadingState />`
   - Add error states with specific handling:
     - LLM unavailable: `t('ai-hub:assistant.llmUnavailable')`
     - Rate limited: `t('ai-hub:assistant.rateLimited')`
     - No budget: `t('ai-hub:assistant.noBudget')`
     - Generic error: `t('ai-hub:assistant.loadError')`
   - Add empty state: `t('ai-hub:assistant.noConversations')` with call-to-action to start new conversation
   - Invalidate conversation list query after successful message send
4. In `AssistantPanel.tsx`:
   - Remove mock fallback logic (the comment mentioning mock fallback)
   - Replace with proper error boundary or error state component
   - Integrate streaming: show typing indicator while streaming, update message content incrementally
5. Add i18n keys to all 4 locale files (en, es, pt-BR, pt-PT):
   - `ai-hub:assistant.loadError`
   - `ai-hub:assistant.noConversations`
   - `ai-hub:assistant.startConversation`
   - `ai-hub:assistant.llmUnavailable`
   - `ai-hub:assistant.rateLimited`
   - `ai-hub:assistant.noBudget`
   - `ai-hub:assistant.sending`
   - `ai-hub:assistant.streaming`
   - `ai-hub:assistant.streamError`
   - `ai-hub:assistant.retryMessage`
6. Verify all existing frontend tests pass
