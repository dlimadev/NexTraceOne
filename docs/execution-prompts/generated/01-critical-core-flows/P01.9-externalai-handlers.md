## 8. Passos obrigatórios
1. Inventory all 8 ExternalAI feature handlers — open each file and classify:
   - **Critical (implement now):** QueryExternalAISimple (health check), QueryExternalAIAdvanced (model selection), GetExternalAIUsage (token tracking), ConfigureExternalAIPolicy (config validation)
   - **Non-critical (documented stub):** ApproveKnowledgeCapture, CaptureExternalAIResponse, ListKnowledgeCaptures, ReuseKnowledgeCapture
2. Implement provider health check (in `QueryExternalAISimple` or create new `CheckProviderHealth` handler):
   - Accept optional `ProviderId` — if null, check all configured providers
   - For each provider: attempt a lightweight connectivity check (e.g., list models endpoint for Ollama, or a simple completion with minimal tokens)
   - Return: provider ID, status (Healthy/Unhealthy/Degraded), latency, last checked timestamp, error message if unhealthy
   - Use `IChatCompletionProvider` or `IExternalAiProviderRepository` to enumerate providers
3. Implement model selection from registry (in `QueryExternalAIAdvanced` or create new handler):
   - Query `AiGovernanceDbContext` Model Registry for available models
   - Replace any references to fictional "NexTrace-Internal-v1" with real model entries from the database
   - Return: model ID, name, provider, capabilities, status (active/inactive), context window size
   - Filter by provider, capability, or active status
4. Implement token usage recording (in `GetExternalAIUsage`):
   - Query persisted token usage data by: conversation ID, user ID, date range, provider, model
   - Aggregate: total tokens, input tokens, output tokens, conversation count, average tokens per conversation
   - Support filtering by tenant and environment
5. Implement provider configuration validation (in `ConfigureExternalAIPolicy`):
   - Accept provider configuration payload
   - Validate: endpoint URL format, required fields present, API key format (if applicable), model name valid
   - Optionally: test connectivity as part of validation
   - Return: validation result with specific field-level errors
6. For non-critical handlers, add clear documentation stub:

7. Verify/extend `IExternalAiProviderRepository` interface if needed for health check and config queries.
8. Add unit tests:

   - Health check returns Unhealthy for unreachable provider (mock)

   - Token usage aggregation is correct
