# PROMPT — Persist AI conversations with token tracking and audit trail

## 1. Identification
- **Prompt ID:** P01.7
- **Phase:** 01-critical-core-flows
- **Execution order:** 9
- **Type:** feature-implement
- **Area:** backend
- **Module:** aiknowledge
- **Bounded context:** ai-orchestration
- **Can run in parallel:** no (depends on P01.6)
- **Risk level:** medium

## 2. Objective
Ensure AI conversations are fully persisted in `AiOrchestrationDbContext` with token tracking per conversation turn and a complete audit trail in `AiGovernanceDbContext`. Create or verify `GetConversation` and `ListConversations` query handlers. Track cumulative token usage per conversation for budget governance.

## 3. Problem actual
Per forensic audit: "AI conversations not persisted. No audit trail for AI interactions despite governance infrastructure existing." The `SendAssistantMessage` handler (after P01.6 fix) will generate real LLM responses, but conversation turns, token usage, and audit entries may not be written to the database. `IAiOrchestrationConversationRepository` exists as an abstraction in `Application/Orchestration/Abstractions/`, but it may not be called from the assistant handler. `AiOrchestrationDbContext` and `AiGovernanceDbContext` exist with migrations, but conversation persistence is not wired end-to-end.

## 4. Resultado esperado
- Every `SendAssistantMessage` call persists the user message and AI response as conversation turns via `IAiOrchestrationConversationRepository`
- Each turn records: role (user/assistant), content, timestamp, token count, model used, provider ID
- Conversations are retrievable by ID and listable by owner
- Cumulative token usage is tracked per conversation for budget governance
- Audit entries are written to `AiGovernanceDbContext` for each AI interaction (who, when, which model, token count, conversation ID)
- `GetConversation` handler returns a complete conversation with all turns
- `ListConversations` handler returns conversations for the current user with summary metadata
