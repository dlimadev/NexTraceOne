# PROMPT — Connect incidents frontend to real API

## 1. Identification
- **Prompt ID:** P01.3
- **Phase:** 01-critical-core-flows
- **Execution order:** 11
- **Type:** feature-implement
- **Area:** frontend
- **Module:** operations
- **Bounded context:** incidents-frontend

## 2. Objective
Replace the hardcoded `mockIncidents` array in `IncidentsPage.tsx` with real API calls using TanStack Query. Connect to the incident list, detail, and correlation endpoints. Add proper loading, error, and empty states with i18n.

## 3. Problem actual
Per forensic audit: "IncidentsPage.tsx uses mockIncidents hardcoded with comment: 'In production will come from API /api/v1/incidents'." The incidents page is one of the two broken core flows. It shows fabricated data that cannot reflect real incidents, correlations, or mitigations.

## 4. Resultado esperado
- IncidentsPage.tsx fetches incident data from `GET /api/v1/incidents`
- Incident detail shows correlated changes from `GET /api/v1/incidents/{id}/correlated-changes`
- TanStack Query manages server state with proper caching
- Loading, error, and empty states are rendered with i18n keys
- Zero references to mockIncidents in the codebase

## 8. Passos obrigatórios
1. Create or update `src/frontend/src/features/operations/api/incidents.ts`:
   - `fetchIncidents()`: GET `/api/v1/incidents` → returns list of incidents
   - `fetchIncidentById(id)`: GET `/api/v1/incidents/{id}` → returns incident detail
   - `fetchCorrelatedChanges(incidentId)`: GET `/api/v1/incidents/{id}/correlated-changes` → returns correlations
   - `triggerCorrelation(incidentId)`: POST `/api/v1/incidents/{id}/correlate` → triggers correlation
   - Use axios instance with auth headers (follow existing patterns in other feature api/ files)
2. In `IncidentsPage.tsx`:
   - Remove the `mockIncidents` array entirely
   - Add `useQuery` from TanStack Query for incident list: `useQuery({ queryKey: ['incidents'], queryFn: fetchIncidents })`
   - Add proper loading state: `if (isLoading) return <LoadingState />`
   - Add error state: `if (error) return <ErrorState message={t('operations:incidents.loadError')} />`
   - Add empty state: `if (!data?.length) return <EmptyState message={t('operations:incidents.noIncidents')} />`
   - Render incident list from real data
3. In incident detail view (if inline or separate component):
   - Add `useQuery` for correlated changes
   - Display correlation list with confidence levels
   - Add "Correlate" button that calls triggerCorrelation
4. Add i18n keys to all 4 locale files:
   - `operations:incidents.loadError`
   - `operations:incidents.noIncidents`
   - `operations:incidents.correlating`
   - `operations:incidents.correlationComplete`
5. Update `incidents.spec.ts`:
   - Replace mock fixtures with API-based fixtures
   - Test that incident list loads from API
   - Test that correlation button triggers API call
6. Verify all existing frontend tests pass
