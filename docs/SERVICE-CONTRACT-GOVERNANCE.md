# SERVICE-CONTRACT-GOVERNANCE.md

## Objective

Ensure all services have well defined contracts.

## Contract Types

-   REST (OpenAPI)
-   SOAP (WSDL/XSD)
-   Kafka events (schemas)
-   Background services

## Required Capabilities

-   Versioning
-   Diff
-   Compatibility checks
-   Approval workflow
-   Examples
-   Ownership
-   AI assisted generation

## Governance Rules

No service without: - Owner - Contract - Version - Documentation
