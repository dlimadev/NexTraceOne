# AI Hub & Governance

## AI Assistant

The **AI Assistant** provides context-aware help for operational tasks:

- Investigate production issues by querying telemetry, incidents, and changes
- Get suggestions for incident root cause analysis
- Ask questions about services, contracts, and operational procedures
- Generate contract drafts based on natural language descriptions

> **Note:** All AI interactions are governed, audited, and subject to your organization's AI policies.

## AI Analysis

The **AI Analysis** page provides AI-powered insights across your platform:
- Anomaly detection in service behavior
- Pattern recognition in incident data
- Optimization recommendations

## Model Registry

Platform Admins can manage available AI models through the **Model Registry**:
- View registered models and their capabilities
- Assign models to specific users, groups, or contexts
- Monitor model usage and performance

## AI Policies

The **AI Policies** page controls how AI is used across the organization:
- Define which models are available to which roles
- Set content and data access restrictions
- Configure prompt guardrails

## AI Routing

**AI Routing** determines which AI model handles each type of request:
- Route different query types to appropriate models
- Configure fallback chains
- Balance cost and quality

## Token Budgets

Manage AI resource consumption through **Token Budgets**:
- Set token quotas per user, team, or role
- Monitor usage against budgets
- Configure alerts for budget thresholds

## AI Audit

The **AI Audit** page provides full transparency into AI usage:
- Complete log of all AI interactions
- Prompt and response audit trail
- Token consumption tracking
- Policy compliance verification

## IDE Integrations

NexTraceOne extends AI capabilities to your development environment:
- Visual Studio and VS Code extensions
- Governed access to NexTraceOne AI from your IDE
- Full audit trail for IDE-based AI interactions
